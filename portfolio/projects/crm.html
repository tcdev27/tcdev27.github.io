Cleversteam: CRM

The Project

Complete

Another Project I have worked on at university is a redesign of a protoype CRM system built by a web development company in liverpool. The front-end of the system utilises bootstrap but the client wants a new design that is more usable and unique.

The Requirements:

The redesign for this CRM needed to tick all of the following:

  • More Unique & Aesthetically Appealing UI.
  • Improved Usability.
  • Better Accessibilty.
  • Responsive Design for Desktop, Tablet & Mobile.
  • No Direct HTML changes, just SASS(css) and CoffeeScript(js).

The CRM now boasts a colour scheme and font-style set that sits closer to that of the client's. On top of this the content is now readable and the interface is usable on both Tablet and Mobile.

Some of the elements within the system were not very accessible so part of the script now adds ARIA Roles (WCAG Guidelines) which make the content compatible with screen readers. In addition all colours used were checked using the WebAim Contrast Checker to ensure a suitable contrast ratio that makes elements visible to those with visual impairments.

In keeping with the client's wishes the HTML of the pages need no direct editing. Instead the provided script will edit it for them.

Methods & Technologies

The Development for this project followed an agile methodology. With a series of small sprints each of which involving the coding to fulfill a selection of the product backlog.

Each iteration involved design(rough sketching), coding and then testing. The end of which sometimes resulted in the addition of new features to the list.

The development was done using the Pre-processors Sass and CoffeeScript which provide efficient CSS and JavaScript coding respectively. The two were compiled into their corresponding languages using Gulp.

Conclusion

The end result of this project is the combination of Styling and Scripting which builds upon the bootstrap framework to create a custom design.

In review, working on top of bootstrap added complexity to the styling that could of been avoided if it had been removed. The removal of it would of also freed up the design to incorporate more creative features.

That said, this build still works perfectly and meets the client's requirements of improving the CRM.
